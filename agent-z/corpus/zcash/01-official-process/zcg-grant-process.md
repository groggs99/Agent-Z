## Overview

ZCG aims to be as transparent as possible. The process enables the Zcash community and ecosystem stakeholders to review and comment on grant applications before approval.

## Before submitting

- Grants exceeding $50,000 USD require KYC (fiduciary may remain pseudonymous to public).
- For non-technical applications, review guidance on the forum.
- Applications accepted on a rolling basis, reviewed every other week.
- The larger the requested sum, the more diligence and milestones required.
- Not sure if ready? Post your idea to the Grants category on the forum (tag @ZcashGrants).

## Submission steps

1. Log into GitHub or create an account.
2. Complete the ZCG Grant application (GitHub issue template) as detailed as possible, ensuring all required fields are complete.
3. Click "Create" to submit.
4. Post a link to the grant on the Zcash Community Forum (required). If unable to post due to new user restrictions, leave a comment below the GitHub issue for posting permissions.
5. FPF (Financial Privacy Foundation) reviews the application for eligibility before ZCG review.

## Community review

- After eligibility review and forum posting, the community and ecosystem stakeholders have at least one week to comment.
- Purpose: community temperature check and crowdsourced due diligence.

## Grant committee review

- The committee may include outside experts.
- The committee reviews the grantee's previous grant outcomes if any.
- Review, discussion, approval decisions, and outstanding questions happen in bi-weekly meetings.

## Finalise application

FPF works with the grantee to improve the application based on committee feedback:
- Adjustments to proposed milestones (deliverables, funding tranches, timeframes)
- More detailed KPIs for grant success
- Further details as required
- The finalised application must meet all requirements.

## Grant committee approval

- Approvals based on a simple majority (3 of 5 committee members).
- Committee provides short reasoning explanations in meeting minutes posted to the forums.
- FPF notifies the grantee via the forums.

## Grant agreement finalised

- FPF facilitates all necessary KYC and grant agreement documentation.
- All required documentation must be completed before milestone payouts.
- Monthly status updates on the forum thread are required before milestone payouts.

## Agent Z process checklist

When helping a user prepare an application, Agent Z should verify:

1. GitHub issue completed with all required fields.
2. Forum post drafted or posted with link back to GitHub issue.
3. Budget reconciles (startup funding + milestones = total requested).
4. KYC flagged if amount exceeds $50,000.
5. Milestones have deliverables, validation criteria, timelines, and payout amounts.
6. Previous grant outcomes referenced if returning applicant.
7. Non-technical guidance followed if applicable.
8. At least one week allowed for community review.

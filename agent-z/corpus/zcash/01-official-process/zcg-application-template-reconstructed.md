## Common terms and conditions observed

Applicants typically confirm:

- agreement to the Grant Agreement terms if funded;
- KYC information if funded above $50,000 USD;
- disclosure of conflicts of interest;
- adherence to ZCG Code of Conduct and Communication Guidelines;
- milestone deliverables will be validated and accepted by intended users or representatives;
- new open-source software will include CONTRIBUTING.md aligned with high Zcash development standards;
- contributions to existing Zcash code will follow project-specific contribution guidelines;
- grant request details will be posted on the Zcash Community Forum;
- forum link will be added to the GitHub issue so community input can happen before ZCG review.

## Agent Z drafting rules

- Always ask for the missing forum link if absent.
- Always validate milestone amounts against total requested amount.
- Always flag KYC if the request exceeds $50,000.
- Always include a clear user/story validation plan for each milestone.
- For open-source work, always include repository, license, CONTRIBUTING.md, tests, documentation, and maintenance plan.
- For non-technical work, always include target audience, proof of reach, itemised budget, timeline, KPIs, and evidence of previous delivery.

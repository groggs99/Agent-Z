# Zenith Full Node Wallet 2025

## Summary

Continuation grant for Zenith, a full-node Zcash wallet supporting TUI and GUI interfaces. Built in Haskell. Proposed adding multi-signature wallet and hardware wallet integration capabilities.

## Zcash alignment

Maps to the wallets wishlist category. FROST/multi-sig component maps directly to a specific wishlist item.

## Problem

The Zcash ecosystem needs full-node wallet options with advanced features like multi-sig and hardware wallet support.

## Proposed solution

Extend Zenith wallet with multi-signature wallets and hardware wallet integration, continuing development of TUI and GUI interfaces.

## Deliverables

Not fully extracted from issue. Likely included: multi-sig implementation, hardware wallet support, GUI improvements.

## Committee decision / status

Declined. Labels show: Grant Declined, Ready For ZCG Review, Grant Application.

## Agent Z lessons

- Returning applicants are not guaranteed approval. Each grant is evaluated independently.
- The wallet category is crowded: Ywallet (Hanh, committee member), Zashi (ECC), and others all compete for the same wishlist slot.
- A wallet grant needs to differentiate sharply: what does this wallet do that no other funded wallet does?
- Full-node wallets face an uphill argument vs light wallets for user-facing applications.
- When Agent Z reviews a wallet proposal, it should check: does this overlap with already-funded wallet work? What is the unique contribution?
